스마틱스 UI Agentation

새로고침 후에도 annotation 툴바를 다시 붙입니다.

북마클릿이나 콘솔 붙여넣기는 한 번만 주입되는 방식이라 새로고침하면 사라집니다.
agent-ui-annotation을 페이지가 열릴 때마다 다시 주입하는 지속형 사용자 스크립트 설치 흐름을 제공합니다.

권장 방식

지속형 사용자 스크립트

생성된 사용자 스크립트를 Tampermonkey 또는 Violentmonkey에 등록하세요. 한 번 설치하면 로컬 개발 서버가 켜져 있는 동안 페이지를 새로고침할 때마다 사이드카가 자동으로 다시 로드됩니다.

사전 준비

  1. Tampermonkey 확장프로그램을 다운로드합니다.
  2. Tampermonkey 확장프로그램의 세부정보에서 사용자 스크립트 허용을 선택해 활성화합니다.
  3. 크롬, 엣지 등의 브라우저를 재시작합니다.

Tampermonkey 설정 방법

  1. 확장프로그램의 대시보드로 이동합니다.
  2. + 버튼을 눌러 새 스크립트를 열고 아래 사용자 스크립트 코드를 복사해서 붙여넣습니다.
  3. 또는 도구를 클릭한 뒤 .user.js를 다운로드해서 가져오기할 수 있습니다.

실행 순서

  1. 처음 한 번 npm install을 실행한 뒤 npm run dev를 실행합니다.
  2. 아래 사용자 스크립트를 복사하거나 .user.js 파일을 다운로드합니다.
  3. 위 방법으로 Tampermonkey에 등록합니다.
  4. 대상 페이지를 새로고침하면 툴바가 자동으로 다시 나타납니다.
.user.js 다운로드

1회성 북마클릿

빠르게 임시 주입이 필요할 때는 기존 북마클릿도 계속 사용할 수 있습니다. 브라우저 확장을 설치하고 싶지 않을 때는 유용하지만, 전체 새로고침 뒤에는 유지되지 않습니다.

북마클릿 실행

CSP가 강한 페이지는 외부 module script 주입을 막을 수 있습니다. 그 경우 이 방식으로는 사이드카가 로드되지 않을 수 있습니다.
